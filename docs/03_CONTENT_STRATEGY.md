# Content Strategy

## Strategy Overview
This document defines the content framework for the website. It intentionally avoids final copy and instead establishes the structure, tone, hierarchy, and future CMS-ready planning for each section.

## Content Principles
- Write for clarity first.
- Keep copy concise, high-trust, and purposeful.
- Use active language and avoid unnecessary jargon.
- Support scanning with clear hierarchy and short paragraphs.
- Prepare all content blocks for future CMS ingestion.

## Global Content Rules
| Rule | Description |
|---|---|
| No fake specifics | Use placeholders for names, numbers, dates, and local references when required. |
| Official tone | Content must feel credible and district-level, not promotional. |
| Strong hierarchy | Every page or section should have one primary message. |
| Modular blocks | Content should be reusable across layouts and devices. |
| Future-ready | Block structure should map cleanly to CMS fields. |

## Placeholder Content Guidelines
Use placeholders whenever real district details are not confirmed:
- [District Representative Name]
- [District Year]
- [Number of Clubs]
- [Event Name]
- [Program Name]
- [Location]
- [Contact Email]

## Future CMS Compatibility
The content model should support:
- Rich text blocks
- Repeatable cards
- Stat groups
- Team profiles
- Event entries
- Media albums
- CTA links
- Section ordering

## Content Governance
Content should be reviewed for:
- Accuracy
- Tone consistency
- Date sensitivity
- Brand alignment
- Accessibility
- SEO clarity
