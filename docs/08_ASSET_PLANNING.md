# Asset Planning

## Overview
This checklist defines the asset categories needed to support a premium district website. Actual assets should be added only when verified and approved.

## Asset Categories

### Logos
| Asset | Purpose | Notes |
|---|---|---|
| Primary district logo | Main brand identifier | Must be high-resolution and official |
| Monogram mark | Compact brand use | Useful for favicon and small surfaces |
| Footer logo | Low-footprint placement | May require simplified treatment |
| Social avatar version | Profile image use | Should remain legible at small sizes |

### Icons
- Navigation icons
- Social icons
- Utility icons
- Contact icons
- Status icons
- Program icons

### Images
| Category | Purpose |
|---|---|
| District photography | Show official activity and community presence |
| Leadership portraits | Support team pages and trust |
| Event photography | Document participation and impact |
| Background imagery | Subtle atmospheric use |
| Archive imagery | Preserve historical records |

### Illustrations
- Editorial illustrations for explainers
- Lightweight decorative graphics
- Map-related visual elements
- Diagram support assets

### Videos
- Hero clips, if ever approved
- Event recap videos
- District highlights
- Social-ready short-form video

### Lottie
- Lightweight motion accents
- Loading states
- Success confirmations
- Subtle illustration motion

### SVGs
- Logo marks
- Dividers and decorative shapes
- Icons and UI symbols
- Map or location graphics

### Photography
| Use Case | Direction |
|---|---|
| Website hero | Wide, high-quality, official imagery |
| Team profiles | Clean portraits with consistent framing |
| Events | Documentary-style coverage |
| Gallery | Organized archival albums |
| Social previews | Strong, compositional images |

### Favicons
- Browser favicon
- Apple touch icon
- PWA icons if introduced later
- Maskable icon for progressive web use

### Social Previews
- Open Graph image
- Twitter/X preview image
- LinkedIn preview image
- Event-specific share artwork

### Fonts
- Geist Sans for the primary interface
- Future heading or display font only if approved
- Fallback system fonts for robustness

### Audio
Audio is not required now, but future possibilities may include:
- Accessibility narration
- Event promo audio
- Media archive sound clips

## Asset Governance
Each asset should be categorized by:
- Source
- Ownership
- Approval status
- Licensing notes
- Recommended dimensions
- Usage contexts

## Asset Checklist
- [ ] Primary logo approved
- [ ] Favicon exported
- [ ] Social previews exported
- [ ] Leadership portraits collected
- [ ] Event archive images organized
- [ ] Brand icons finalized
- [ ] Illustration direction approved
- [ ] OG assets verified

## Naming Conventions
Use descriptive, consistent names such as:
- `district-logo-primary.svg`
- `district-avatar.png`
- `event-2026-recap-01.jpg`
- `og-home.jpg`

## Future Asset Management
A future DAM or CMS integration should support metadata such as:
- Title
- Alt text
- Photographer
- License
- Category
- Event association
- Usage restrictions
